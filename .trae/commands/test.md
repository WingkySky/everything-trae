# 测试生成 (Test Generation)

## 命令
`/test`

## 功能描述
为指定的代码生成完整的测试用例，包括单元测试、边界测试和异常测试。

## 使用场景
- 新功能开发
- Bug 修复
- 代码重构
- 测试覆盖率提升

## 测试类型
1. **单元测试**
   - 正常路径测试
   - 边界条件测试
   - 异常处理测试

2. **集成测试**
   - 模块间交互测试
   - 数据流测试
   - 接口契约测试

3. **性能测试**
   - 响应时间测试
   - 并发压力测试
   - 资源监控

## 输出格式
- 测试文件与源码对应
- 测试用例命名清晰
- 包含详细的注释
- 提供测试报告

## 示例
```
用户: /test src/utils/validator.ts
AI: [生成完整的测试文件]
```

## 注意事项
- 覆盖所有边界条件
- 测试用例要独立
- 遵循项目测试规范
