# 代码审查技能

## 触发场景
当用户提交代码进行审查、重构代码或修复 Bug 后，自动触发代码审查流程。

## 审查清单

### 代码质量
- [ ] 代码是否符合项目规范
- [ ] 是否有重复代码
- [ ] 函数复杂度是否合理（圈复杂度 ≤ 10）
- [ ] 命名是否清晰准确
- [ ] 代码是否易于理解

### 安全性
- [ ] 是否存在注入风险
- [ ] 敏感信息是否泄露
- [ ] 输入验证是否充分
- [ ] 权限控制是否完善
- [ ] 错误信息是否泄露敏感数据

### 性能
- [ ] 是否存在性能瓶颈
- [ ] 是否有不必要的计算
- [ ] 数据库查询是否优化
- [ ] 缓存策略是否合理
- [ ] 是否有内存泄漏风险

### 可维护性
- [ ] 代码是否模块化
- [ ] 是否有充分的注释
- [ ] 依赖管理是否规范
- [ ] 错误处理是否完善
- [ ] 日志记录是否合理

### 测试覆盖
- [ ] 是否有单元测试
- [ ] 测试覆盖率是否达标
- [ ] 测试是否覆盖边界条件
- [ ] 是否有集成测试

## 审查流程

### 1. 代码检查
- 阅读代码实现
- 检查是否符合规范
- 识别潜在问题

### 2. 问题分类
- **严重**：安全漏洞、逻辑错误
- **警告**：性能问题、可维护性问题
- **建议**：代码风格、最佳实践

### 3. 反馈输出
- 按严重程度分类问题
- 提供具体位置和修复建议
- 总结整体代码质量

## 输出规范
- 使用 Markdown 格式
- 问题编号和分类
- 具体位置（文件名、行号）
- 修复建议和示例代码

## 使用示例
- "审查这段代码"
- "检查这个 PR 的代码质量"
- "重构后进行代码审查"
