# Git 提交规范技能

## 触发场景
当用户执行 git commit 操作时，自动检查提交信息是否符合规范。

## 提交格式

### 基本格式
```
<type>(<scope>): <subject>
```

### 类型说明
- `feat`：新功能（feature）
- `fix`：修复 Bug
- `docs`：文档更新
- `style`：代码格式调整（不影响代码运行）
- `refactor`：重构
- `test`：测试相关
- `chore`：构建过程或辅助工具变动

### 范围说明
- 指定变更影响的模块或文件
- 如：`feat(user): 添加用户登录功能`
- 如：`fix(api): 修复订单查询接口`

### 主题要求
- 使用祈使句，不超过 50 个字符
- 首字母小写，不以句号结尾
- 描述做了什么

## 详细格式

### 完整格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Body 部分
- 详细说明变更内容
- 说明变更原因
- 列出关键改动点

### Footer 部分
- 关联 Issue：`Fixes #123`
- Breaking Change：`BREAKING CHANGE: ...`
- 其他说明

## 示例

### 新功能
```
feat(user): 添加用户登录功能

- 实现 JWT 认证
- 添加密码加密
- 支持第三方登录

Fixes #123
```

### Bug 修复
```
fix(api): 修复订单查询接口超时问题

- 优化数据库查询
- 添加索引
- 调整查询逻辑

Fixes #456
```

### 重构
```
refactor(auth): 重构认证模块

- 提取认证逻辑到独立模块
- 优化中间件顺序
- 添加单元测试

BREAKING CHANGE: 认证中间件签名变更
```

## 检查清单
- [ ] 提交信息格式正确
- [ ] 主题不超过 50 字符
- [ ] 类型使用正确
- [ ] 有详细说明（如需要）
- [ ] 关联 Issue（如需要）
