# 技术文档编写规范技能

## 触发场景
当用户需要编写或更新技术文档时，自动应用文档规范。

## 文档类型

### API 文档
- 端点说明
- 请求/响应格式
- 错误码说明
- 使用示例

### 开发文档
- 环境搭建
- 项目结构说明
- 开发规范
- 常见问题

### 用户文档
- 功能介绍
- 使用指南
- 最佳实践
- 故障排查

### 架构文档
- 系统架构图
- 模块关系说明
- 技术选型理由
- 部署方案

## 文档结构

### 基本结构
```
# 文档标题

## 目录

## 1. 概述
- 背景介绍
- 目标说明

## 2. 快速开始
- 环境要求
- 安装步骤
- 基本示例

## 3. 详细说明
- 功能介绍
- 使用方法
- 参数说明

## 4. 示例
- 示例 1
- 示例 2

## 5. 常见问题
- 问题 1
- 问题 2

## 6. 总结
```

## 内容规范

### 语言要求
- 使用简洁准确的语言
- 避免歧义和模糊表达
- 保持一致的术语

### 示例要求
- 示例代码完整可运行
- 标注关键代码
- 提供预期输出

### 格式要求
- 使用标准 Markdown
- 代码块标注语言
- 重要概念加粗
- 链接指向准确

## 图表规范
- 架构图清晰完整
- 流程图步骤明确
- 图表有编号和标题
- 图表有文字说明

## 更新规范
- 版本号管理
- 更新日志记录
- 废弃功能标注
- 文档定期审查

## 输出要求
- 文档结构清晰
- 示例完整可执行
- 语言通俗易懂
- 包含常见问题
