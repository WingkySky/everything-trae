# 代码规范规则

## 文件命名规范
- 组件文件使用 PascalCase，如 `UserProfile.tsx`
- 工具函数文件使用 camelCase，如 `user-utils.ts`
- 常量文件使用 UPPER_SNAKE_CASE，如 `API_URLS.ts`

## 代码结构规范
- 函数长度不超过 50 行
- 类的复杂度不超过 100 行
- 文件长度不超过 500 行
- 函数参数不超过 5 个

## 注释规范
- 公共函数必须添加注释
- 复杂逻辑必须添加注释
- 注释要说明"为什么"而不是"是什么"
- 注释要保持更新

## 错误处理规范
- 所有异步操作必须处理错误
- 错误信息要清晰明确
- 不要吞掉错误
- 使用自定义错误类型

## 性能规范
- 避免不必要的计算
- 使用虚拟滚动处理长列表
- 优化数据库查询
- 实现适当的缓存策略
