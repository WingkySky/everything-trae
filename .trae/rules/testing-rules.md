# 测试规则

## 测试覆盖率
- 单元测试覆盖率 ≥ 80%
- 关键业务逻辑覆盖率 ≥ 90%
- 边界条件覆盖率 ≥ 85%
- 异常处理覆盖率 ≥ 80%

## 测试类型
- 单元测试：测试单个函数/组件
- 集成测试：测试模块间交互
- E2E 测试：测试完整业务流程
- 性能测试：测试性能指标

## 测试编写
- 测试用例要独立
- 测试用例要可重复
- 测试用例要清晰
- 测试用例要完整

## 测试运行
- 提交前运行所有测试
- PR 前运行所有测试
- 发布前运行所有测试
- 定期运行性能测试

## 测试工具
- 使用 Jest 进行单元测试
- 使用 React Testing Library 进行组件测试
- 使用 Cypress 进行 E2E 测试
- 使用 Jest Performance 进行性能测试

## 测试命名
- 测试文件名：`*.test.ts`
- 测试函数名：`describe` 和 `it`
- 测试描述要清晰
- 测试断言要明确
