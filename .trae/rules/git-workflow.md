# Git 工作流规则

## 分支策略
- 主分支：`main` 用于生产环境
- 开发分支：`develop` 用于集成测试
- 功能分支：`feature/xxx` 用于新功能开发
- 修复分支：`fix/xxx` 用于 Bug 修复
- 发布分支：`release/xxx` 用于发布准备

## 提交规范
- 遵循 Conventional Commits 规范
- 提交信息要清晰明确
- 每次提交只做一件事
- 使用英文提交（团队约定）

## PR 规范
- PR 要小而聚焦
- PR 必须通过代码审查
- PR 必须通过所有测试
- PR 必须更新相关文档

## 合并策略
- 使用 squash merge 合并 PR
- 合并前要解决所有冲突
- 合并后要更新版本号
- 合并后要删除分支

## 标签规范
- 使用语义化版本
- 标签格式：`v1.0.0`
- 发布前要打标签
- 标签要附带发布说明

## 回滚策略
- 发布前要备份
- 回滚要快速简单
- 回滚后要记录原因
- 回滚后要创建修复分支
