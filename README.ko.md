# Everything Trae

Trae AI 에이전트 설정 모음 - 에이전트 성능 향상

## 개요

이 프로젝트는 [everything-claude-code](https://github.com/affaan-m/everything-claude-code) 프로젝트의 성공적인 경험을 참고하여, Trae AI 프로그래밍 도구를 위한 완전한 설정 솔루션을 제공합니다. 전문적인 에이전트, 스킬 모듈, 자동화 후크, 슬래시 명령어, 룰, 외부 도구 통합(MCP)을 설정하여, Trae를 일반적인 AI 어시스턴트에서 전문적인 개발 팀으로 변화시킵니다.

## 핵심 기능

### 1. 전문 에이전트 (10)

6개의 전문 에이전트를 설정했습니다：

- **planner** - 플래너：요구사항 분석, 기술 선택, 작업 분해
- **coder** - 코드 생성：코드 구현, 문서 작성, 테스트 생성
- **reviewer** - 코드 리뷰：품질 평가, 보안 검사, 성능 최적화
- **debugger** - 디버깅 전문가：문제 특정, 수정 방안, 테스트 검증
- **tester** - 테스트 전문가：테스트 전략, 테스트 케이스 작성, 커버리지 분석
- **writer** - 문서 전문가：API 문서, 개발 문서, 아키텍처 문서
- **architect** - 아키텍트：시스템 아키텍처 설계, 기술 선택 평가, 아키텍처 리뷰
- **devops** - DevOps 전문가：CI/CD 파이프라인, Infrastructure as Code, 컨테이너화
- **performance** - 성능 전문가：성능 분석, 최적화 솔루션, 벤치마크 테스트
- **database** - 데이터베이스 전문가：데이터베이스 설계, SQL 최적화, 성능 튜닝

### 2. 스킬 모듈 (8)

4개의 핵심 스킬 모듈을 설정했습니다：

- **tdd-workflow** - 테스트 주도 개발 워크플로우
- **code-review-standards** - 코드 리뷰 기준 및 모범 사례
- **git-commit-standards** - Git 커밋 사양
- **documentation-standards** - 기술 문서 작성 기준
- **frontend-architecture** - 프론트엔드 아키텍처 설계（컴포넌트, 상태 관리, 라우팅）
- **api-design** - API 설계（RESTful, GraphQL, API 문서）
- **microservices** - 마이크로서비스 아키텍처（서비스 분할, 통신, 거버넌스）

### 3. 자동화 후크 (4)

4개의 자동화 후크를 설정했습니다：

- **pre-commit** - 커밋 전 후크：코드 검사, 테스트 검증, 보안 스캔
- **post-commit** - 커밋 후 후크：알림, 통계, 정리
- **pre-push** - 푸시 전 후크：코드 검사, 빌드 검증, 준수 검사
- **build** - 빌드 후크：준비, 실행, 검증

### 4. 슬래시 명령어 (6)

6개의 실용적인 슬래시 명령어를 설정했습니다：

- **/plan** - 플랜 작업：코드 작성 전에 개발 계획 생성
- **/review** - 코드 리뷰：포괄적인 코드 품질 및 보안 리뷰
- **/test** - 테스트 생성：코드의 완전한 테스트 케이스 생성
- **/docs** - 문서 생성：코드의 완전한 문서 생성
- **/optimize** - 코드 최적화：성능 최적화, 구조 최적화, 모범 사례
- **/security** - 보안 감사：포괄적인 보안 리뷰 및 취약점 발견

### 5. 룰 (3)

3개의 핵심 룰 파일을 설정했습니다：

- **coding-standards** - 코드 사양（파일 네이밍, 코드 구조, 주석, 오류 처리）
- **security-rules** - 보안 룰（입력 검증, 인증, 데이터 보안, API 보안）
- **testing-rules** - 테스트 룰（테스트 커버리지, 테스트 유형, 테스트 작성, 테스트 실행）

### 6. 외부 도구 통합（MCP） (8)

8개의 MCP 서버를 설정했습니다：

- **filesystem** - 파일 시스템 작업
- **git** - Git 버전 관리
- **memory** - 영속 메모리
- **github** - GitHub 저장소 작업
- **postgres** - PostgreSQL 데이터베이스 작업
- **browser** - 브라우저 자동화
- **sentry** - Sentry 오류 모니터링
- **slack** - Slack 메시지 알림

## 빠른 시작

### 설치

1. `.trae` 디렉토리를 프로젝트 루트 디렉토리에 복사하세요
2. Trae가 모든 설정을 자동으로 로드합니다
3. AI 프로그래밍 어시스턴트 사용을 시작하세요

### 사용 방법

1. **적절한 에이전트 선택**
   ```
   @planner 이 요구사항을 분석하고 개발 계획을 작성해주세요
   @coder 이 기능을 구현해주세요
   @reviewer 이 코드를 리뷰해주세요
   @architect 시스템 아키텍처를 설계해주세요
   @devops CI/CD 파이프라인을 설정해주세요
   @performance 성능 병목을 분석해주세요
   @database 데이터베이스 쿼리를 최적화해주세요
   ```

2. **슬래시 명령어 사용**
   ```
   /plan        - 플랜 작업
   /review      - 코드 리뷰
   /test        - 테스트 생성
   /docs        - 문서 생성
   /optimize    - 코드 최적화
   /security    - 보안 감사
   ```

3. **스킬 트리거**
   - TDD 개발：요구사항을 설명하면 Trae가 자동으로 TDD 워크플로우를 사용합니다
   - 코드 리뷰：제출 전 자동으로 리뷰 프로세스가 트리거됩니다
   - Git 커밋：표준화된 커밋 메시지 형식을 따릅니다

4. **MCP 도구 사용**
   - 파일 작업：Trae가 프로젝트 파일의 읽기 및 쓰기를 할 수 있습니다
   - Git 작업：Trae가 Git 저장소의 보기 및 조작을 할 수 있습니다
   - 데이터베이스 작업：Trae가 PostgreSQL 데이터베이스와 상호 작용할 수 있습니다
   - 브라우저 자동화：Trae가 브라우저 작업을 자동화할 수 있습니다
   - 오류 모니터링：Trae가 Sentry 오류 보고서에 접근할 수 있습니다
   - 메시지 알림：Trae가 Slack에 메시지를 보낼 수 있습니다

## 모범 사례

### 1. 에이전트 사용

- 작업 유형에 따라 적절한 에이전트를 선택
- 작업 요구사항을 명확히 설명
- 시의적절한 피드백 및 조정 제공

### 2. 스킬 사용

- 기존 스킬 재사용
- 프로젝트 전용 스킬 생성
- 정기적으로 스킬 업데이트

### 3. 후크 사용

- 필요한 후크 설정
- 후크 실행 시간 최적화
- 후크 실행 상태 모니터링

### 4. 슬래시 명령어 사용

- 슬래시 명령어를 사용하여 빠른 작업 시작
- 컨텍스트 전환 불필요
- 복잡한 작업은 에이전트와 조합

### 5. 룰 사용

- 룰은 자동으로 적용됩니다
- 프로젝트 전체에서 일관성 보장
- 프로젝트의 진화에 따라 룰 업데이트

### 6. MCP 사용

- 필요한 MCP 서버만 활성화
- MCP 액세스 권한 제한
- 정기적으로 MCP 설정 검토

## 문제 해결

### 일반적인 문제

1. **설정이 적용되지 않음**
   - 설정 파일 경로 확인
   - 설정 파일 형식 검증
   - Trae 로그 확인

2. **성능 문제**
   - 에이전트 수 감소
   - 스킬 크기 최적화
   - MCP 서버 수 감소

3. **권한 문제**
   - 파일 권한 확인
   - MCP 설정 검증
   - 사용자 권한 확인

4. **검증 오류**
   - `./validate-config.sh`를 실행하여 설정을 확인
   - 오류 메시지 확인
   - 설정 파일을 적절히 업데이트
